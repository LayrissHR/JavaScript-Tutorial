Урок: Променливи в JavaScript

Как компютърът "помни" неща

Какво е променлива?

Най-просто казано, променливата е кутия с етикет, в която съхраняваме информация. Когато пишем код, ние постоянно трябва да помним разни неща: името на потребителя, брой точки в игра, текуща дата и т.н. Променливите ни позволяват да дадем име на тази информация и да я използваме по-късно.

Процесът има две стъпки:

  1. Декларация (Declaration): Казваме на JavaScript: "Хей, създай ми нова празна кутия и ѝ сложи етикет 'име'".
  2. Присвояване (Assignment): Казваме: "В кутията с етикет 'име' искам да сложиш стойността 'Иван'".

Можем да направим и двете на един ред:

Кодът:

let myName = "Петър";
console.log(myName); // Ще изпише "Петър" в конзолата

myName = "Георги"; // Можем да сменим стойността
console.log(myName); // Ще изпише "Георги"

Обяснение ред по ред:

  • let myName = "Петър";: let е ключовата дума за декларация. myName е етикетът (името) на нашата "кутия". = е операторът за присвояване. "Петър" е стойността, която слагаме в кутията.
  • console.log(myName);: Показваме в конзолата това, което в момента се намира в кутията myName.
  • myName = "Георги";: Тъй като кутията вече съществува, не е нужно да пишем let. Просто присвояваме нова стойност. Старата ("Петър") се изхвърля.

Правила за именуване на променливи

Когато давате "етикет" на вашата "кутия", трябва да спазвате няколко правила. Името на променливата:

  • Може да започва само с буква, долар ($) или долна черта (_). Не може да започва с число.
  • Може да съдържа букви, числа, долари и долни черти.
  • Е чувствително към регистъра (case-sensitive). Това означава, че myName и myname са две напълно различни променливи.
  • Не може да бъде някоя от "резервираните думи" на JavaScript (let, if, for, function и т.н.).

Най-добра практика: camelCase

В JavaScript (и много други езици) е прието променливите да се именуват със стил, наречен "camelCase" (камилски регистър). Първата дума е с малка буква, а всяка следваща започва с главна, без интервали.

Примери: let myVariableName;, const userAge;, let scoreInGame;

Трите ключови думи: var, let и const

В JavaScript има три начина да "създадем кутия" (да декларираме променлива). Всеки има своите особености.

1. let (Модерният стандарт)

Това е предпочитаният начин в модерния JavaScript. Променливите, създадени с let, могат да променят стойността си и имат т.нар. блоков обхват (block scope). Това означава, че те съществуват само в рамките на къдравите скоби { ... }, в които са създадени.

2. const (За константи)

Използвайте const, когато знаете, че стойността никога няма да се променя. Това прави кода по-сигурен и лесен за разбиране. Опитът да промените const променлива ще доведе до грешка. 

const pi = 3.14159;
pi = 3; // Това ще даде грешка!

Важно: Ако const е обект или масив, вие можете да променяте съдържанието му, но не можете да му присвоите изцяло нов обект/масив.

3. var (Старият начин)

Това е оригиналният начин за деклариране на променливи. var има функционален обхват (function scope), което означава, че "живее" в цялата функция, независимо от { ... } блоковете. Това често води до обърквания и грешки.

Защо да избягваме var?

В днешно време се препоръчва винаги да използвате let и const. var има странно поведение, наречено "hoisting" (издигане), което може да направи кода ви непредсказуем. С let и const кодът ви е по-чист и по-малко податлив на грешки.

Основни типове данни

JavaScript е "динамично типизиран" език. Това означава, че не е нужно да казваме на "кутията" какъв тип информация ще държи. Тя сама се адаптира. 

let myVar = "Здрасти!"; // myVar е String (текст)
myVar = 100;         // Сега myVar е Number (число)
myVar = true;        // А сега е Boolean (логическо)

Ето основните типове данни, които трябва да знаете:

  • String (Низ): Текст, винаги в кавички. ("Здрасти", '123')
  • Number (Число): Всякакви числа. (10, 3.14, -50)
  • Boolean (Булев): Логически тип. Може да бъде само true (вярно) или false (невярно).
  • Object (Обект): Колекция от данни. ({ име: "Иван", възраст: 30 })
  • Array (Масив): Специален вид обект - списък с данни. ([ "ябълки", "банани", "череши" ])
  • undefined: Стойността на "празна" кутия, която е декларирана, но нищо не ѝ е присвоено.
  • null: Представлява нарочно зададена "липса на стойност".

Шаблонни низове (Template Literals)

Това е модерен и по-удобен начин за работа с низове, който използва (`) (обратни кавички) вместо обикновени. Те позволяват лесно да "вграждаме" променливи в текста, използвайки синтаксиса ${...}. 

const name = "Мария";
const age = 28;

// Старият начин:
const greetingOld = "Здравей, " + name + "! Ти си на " + age + " години.";

// Новият начин (Template Literal):
const greetingNew = \`Здравей, ${name}! Ти си на ${age} години.\`;

console.log(greetingNew); // Ще изпише същото

Автоматично преобразуване на типове (Type Coercion)

JavaScript често се опитва да бъде "услужлив" и автоматично преобразува типовете данни, когато извършвате операции с тях. Това може да бъде колкото полезно, толкова и объркващо. 

let result1 = 5 + "5";   // "5" е низ, JS превръща 5 в низ
// Резултат: "55" (низ)

let result2 = 10 - "5";  // "5" е низ, но '-' е само за числа
                         // JS превръща "5" в число
// Резултат: 5 (число)

Внимание!

Това автоматично преобразуване е една от най-големите "странности" на JavaScript. Винаги бъдете наясно с какъв тип данни работите, за да избегнете неочаквани резултати. Когато събирате стойности от HTML полета, те винаги идват като низове (string)!

Време за практика! Задачи с променливи

Кликнете върху всяка задача, за да видите условието.

1. Декларация
Декларирайте променлива с име myName и ѝ присвоете вашето име.
2. Константа
Декларирайте константа с име yearOfBirth и ѝ присвоете вашата година на раждане. Опитайте да ѝ присвоите нова стойност и вижте каква грешка дава конзолата.
3. Събиране на числа
Създайте две променливи, num1 и num2, и им присвоете стойности 10 и 20. Създайте трета променлива sum, която е равна на сбора им. Изпишете sum в конзолата.
4. Свързване на низове (Template Literal)
Създайте променливи firstName и lastName. Създайте трета променлива fullName, като използвате template literal (с обратни кавички ``), за да ги съедините с интервал по средата.
5. Булева логика
Създайте променлива isLearning и ѝ присвоете стойност true. Създайте променлива isTired и ѝ присвоете false.
6. Размяна на стойности
Създайте две променливи: let a = 5; и let b = 10;. Без да пишете нови числа, а само с помощта на трета, временна променлива (let temp), разменете стойностите им, така че a да стане 10, а b да стане 5.
7. Създаване на обект
Създайте обект (const car) с три свойства: make (марка), model (модел) и year (година).
8. Създаване на масив
Създайте масив с име shoppingList, който съдържа 3 ваши любими продукта за пазаруване.
9. Type Coercion проблем
Какъв ще е резултатът от "10" + 20? А от "10" - 5? Проверете в конзолата и обяснете защо се получава така.
10. Изчисляване на възраст
Създайте константа currentYear с текущата година. Използвайки променливата yearOfBirth от задача 2, създайте нова променлива age, която изчислява вашата възраст.
