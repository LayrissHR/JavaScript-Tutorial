Какво е условен оператор?

Досега нашият код се изпълняваше ред по ред, отгоре надолу, без да се замисля. Условните оператори ни позволяват да променим това. Те са като кръстопът за нашия код. Ние му задаваме въпрос (условие) и ако отговорът е "Да" (true), той тръгва по един път (изпълнява един код), а ако е "Не" (false), тръгва по друг.

Въпросът, който задаваме, винаги трябва да има отговор "Да" или "Не" (true или false). Например: "По-голямо ли е числото 5 от 3?" (Да), "Равно ли е името на потребителя на 'Иван'?" (Може би да, може би не).

Основата: if, else if, и else

Това е най-често срещаната и важна условна конструкция. Тя се чете почти като на български: "Ако..., в противен случай ако..., в противен случай..."

1. if (Ако)

Изпълнява код само ако условието в скобите () е вярно (true). 

let age = 20;

if (age >= 18) {
  console.log("Вие сте пълнолетен!");
}
// Резултат: "Вие сте пълнолетен!"

Обяснение ред по ред:

  • let age = 20;: Създаваме променлива age със стойност 20.
  • if (age >= 18) { ... }: "Ако стойността в age (20) е по-голяма или равна на 18..."
  • console.log(...): Тъй като 20 е по-голямо от 18 (true), кодът в къдравите скоби { } се изпълнява.

2. if ... else (Ако ... в противен случай)

else улавя всички случаи, в които if условието е било false. 

let age = 15;

if (age >= 18) {
  console.log("Вие сте пълнолетен!");
} else {
  console.log("Вие сте непълнолетен.");
}
// Резултат: "Вие сте непълнолетен."

Обяснение:

Тук age е 15. Условието (15 >= 18) е false. Затова кодът в if блока се пропуска и се изпълнява кодът в else блока.

3. if ... else if ... else (Верига от проверки)

Позволява ни да проверяваме няколко условия едно след друго.

let temperature = 15;

if (temperature > 25) {
  console.log("Времето е горещо, облечете тениска.");
} else if (temperature > 10) {
  console.log("Времето е приятно, вземете си яке.");
} else {
  console.log("Времето е студено, облечете се дебело.");
}
// Резултат: "Времето е приятно, вземете си яке."

Обяснение:

  • Проверява (15 > 25). Това е false. Пропуска if блока.
  • Преминава към else if. Проверява (15 > 10). Това е true!
  • Изпълнява кода в else if блока и изписва "Времето е приятно...".
  • Тъй като е намерил вярно условие, той игнорира следващия else блок и излиза от цялата конструкция.

Интерактивен "Пропуск за клуба":

Оператори за сравнение и логически оператори

За да "задаваме въпроси" в нашите if-ове, ние се нуждаем от специални оператори. Ето най-важните от тях в една таблица:

Оператор Име Обяснение
== Равно (с преобразуване) Проверява дали стойностите са равни (напр. 5 == "5" е true).
=== Строго равно Проверява дали стойностите И типовете са равни (напр. 5 === "5" е false).
!= Неравно (с преобразуване) Проверява дали стойностите са различни.
!== Строго неравно Проверява дали стойностите ИЛИ типовете са различни.
> По-голямо от Проверява дали лявата стойност е по-голяма от дясната.
< По-малко от Проверява дали лявата стойност е по-малка от дясната.
>= По-голямо или равно Проверява дали лявата стойност е по-голяма или равна на дясната.
<= По-малко или равно Проверява дали лявата стойност е по-малка или равна на дясната.
&& Логическо "И" (AND) Връща true, само ако и двете условия отляво и отдясно са верни.
|| Логическо "ИЛИ" (OR) Връща true, ако поне едно от условията е вярно.
! Логическо "НЕ" (NOT) Обръща стойността: !true става false, а !false става true.

Най-важното правило: == срещу ===

Това е най-голямата уловка за начинаещи в JavaScript. Операторът == се опитва да преобразува типовете, преди да ги сравни, което води до странни резултати: 

console.log(5 == "5"); // Резултат: true (странно!)

Операторът === (с три знака "равно") е стриктен. Той проверява дали стойностите и техните типове са напълно еднакви: 

console.log(5 === "5"); // Резултат: false (логично!)

Златно правило: Винаги използвайте === и !==, освен ако нямате много основателна причина да не го правите. 

let age = 25;
let hasTicket = true;

// && (И)
if (age > 18 && hasTicket === true) {
  console.log("Може да влезете."); // Ще се изпълни
}

// || (ИЛИ)
let isRaining = false;
let isSnowing = true;
if (isRaining === true || isSnowing === true) {
    console.log("Вземете си чадър или шал."); // Ще се изпълни
}

// ! (НЕ)
let isLoggedIn = false;
if (!isLoggedIn) { // Чете се: "Ако НЕ Е логнат"
    console.log("Моля, влезте в профила си."); // Ще се изпълни
}

Алтернативата: switch (Превключвател)

Понякога имаме дълга верига от else if, която проверява една и съща променлива за различни точни стойности. В такъв случай, switch е по-четим и подреден начин.

Представете си го като разпределително табло. Подавате му една променлива и то "превключва" към правилния "бушон" (case).

Интерактивен "Дни от седмицата":

Кодът зад примера:

let day = "Сряда"; // Взето от полето
let activity;

switch (day) {
  case "Понеделник":
    activity = "Начало на работната седмица.";
    break; // break е КРИТИЧНО! Спира изпълнението.
  case "Вторник":
  case "Сряда":
  case "Четвъртък":
    activity = "Работа и учене.";
    break;
  case "Петък":
    activity = "Почти е уикенд!";
    break;
  case "Събота":
  case "Неделя":
    activity = "Време за почивка!";
    break;
  default:
    activity = "Моля, въведете валиден ден.";
}
// activity ще бъде "Работа и учене."

Обяснение ред по ред:

  • switch (day) { ... }: Започваме конструкцията. Ще проверяваме променливата day.
  • case "Понеделник":: "В случай, че day е равно на 'Понеделник'..."
  • activity = ...: Присвояваме стойност.
  • break;: Най-важната част! Казва на switch-а: "Спри дотук, свършихме работа!". Ако го пропуснем, кодът ще продължи да се изпълнява и в следващия case!
  • case "Вторник": case "Сряда":...: Можем да групираме няколко case-а, за да изпълнят един и същ код.
  • default:: Това е като else. Изпълнява се, ако нито един от case-овете не е бил верен.

Време за практика! Задачи с условия

Кликнете върху всяка задача, за да видите условието.

1. Четно или нечетно?
Създайте променлива number. Напишете if-else, което да изписва в конзолата "Числото е четно" или "Числото е нечетно". (Подсказка: Използвайте оператора за остатък %. number % 2 === 0 е вярно, ако числото е четно).
2. Оценка с думи
Създайте променлива grade (число от 2 до 6). Напишете верига от if-else if-else, която изписва "Слаб", "Среден", "Добър", "Мн. Добър" или "Отличен" спрямо оценката.
3. Логин форма
Създайте две променливи: let username = "admin"; и let password = "12345";. Напишете if с &&, който проверява дали И потребителското име, И паролата са верни, и изписва "Добре дошъл, admin!".
4. Гост-потребител
Създайте променлива let userRole = "guest";. Напишете if с ||, което проверява дали userRole е "admin" ИЛИ "editor" и изписва "Имате права за редактиране".
5. Светофар
Създайте променлива let lightColor = "yellow";. Напишете switch конструкция, която за "red" изписва "Спри!", за "yellow" - "Внимание!", а за "green" - "Премини!".
6. Тернарен оператор (за напреднали)
Тернарният оператор е кратък if-else. Синтаксисът е: условие ? ако_е_true : ако_е_false;. Опитайте да пренапишете задача 1 (четно/нечетно) с тернарен оператор.
7. Проверка за `null` или `undefined`
Създайте променлива let name; (без стойност). Напишете if, което проверява if (name === null || name === undefined) и изписва "Липсва име".
8. `switch` с числа
Пренапишете задача 2 (оценките) с помощта на switch. (Подсказка: switch (true) { case (grade === 2): ... } е трик, който работи, но помислете дали е по-четимо от if-else).
9. Проверка на дължина на име
Създайте променлива username. Напишете if-else, което проверява дали дължината на името (username.length) е по-малка от 5 символа. Ако е, да изпише "Името е твърде кратко". В противен случай, "Името е прието".
10. "Falsy" стойности
Създайте променлива let userData = 0;. Напишете if (userData) { ... } else { ... }. В if-a изпишете "Потребителят има данни", а в else-a "Липсват данни". Какъв е резултатът? Опитайте със "", null, undefined и false. Това са "Falsy" стойности, които if-ът третира като false.
