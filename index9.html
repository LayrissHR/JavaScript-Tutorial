Урок: Валидация на формуляри

Как да защитим данните си от грешки

Защо ни е нужна валидация?

Валидацията е процесът на проверка дали данните, въведени от потребителя, отговарят на определени правила, преди да бъдат изпратени към сървъра. Това е първата линия на защита.

Представете си, че поръчвате пица онлайн, но забравите да напишете адреса си. Ако сайтът не ви спре (валидира), поръчката ще тръгне, но пицата никога няма да пристигне. Валидацията спестява време и нерви на всички.

Клиентска vs Сървърна валидация

Ние тук учим клиентска валидация (в браузъра). Тя е за удобство на потребителя. Но не забравяйте: един хакер може да заобиколи браузъра! Затова данните винаги трябва да се проверяват отново и на сървъра.

Статична срещу Динамична проверка

Това са двата основни подхода, които често се комбинират.

1. Статична валидация (При изпращане)

Проверката се извършва чак когато потребителят натисне бутона "Изпрати" (Submit).

  • Плюс: Не досажда на потребителя, докато пише.
  • Минус: Потребителят може да се изненада от 10 грешки наведнъж накрая.
form.addEventListener('submit', function(event) {
  // 1. Спираме изпращането
  event.preventDefault(); 
  
  // 2. Проверяваме
  if (input.value === "") {
     alert("Грешка!");
  } else {
     form.submit(); // Ръчно изпращане, ако всичко е точно
  }
});

2. Динамична валидация (В реално време)

Проверката се извършва моментално, докато потребителят пише или когато напусне полето. Използваме събитията input (при всяка буква) или blur (при излизане от полето).

  • Плюс: Веднага виждаш дали си сбъркал.
  • Минус: Ако е твърде агресивна (напр. да ти казва "Невалиден имейл" още докато си написал само първата буква), може да е дразнещо.
// Слушаме за всяка промяна в полето
usernameInput.addEventListener('input', function() {
  if (usernameInput.value.length < 3) {
     errorMsg.style.display = "block";
     usernameInput.style.borderColor = "red";
  } else {
     errorMsg.style.display = "none";
     usernameInput.style.borderColor = "green";
  }
});

Супер-силата: Регулярни изрази (Regex)

Понякога трябва да проверим дали текстът следва сложен шаблон (например: дали е валиден имейл, телефон или ЕГН). Тук на помощ идват Регулярните изрази. Те са като "код за търсене на текст".

В JavaScript те се ограждат с наклонени черти /.../. Използваме метода .test(), за да проверим дали текстът отговаря на шаблона.

  • /^\d+$/ - Само цифри (от началото ^ до края $).
  • /[a-z]/ - Съдържа поне една малка буква.
  • /^[a-zA-Z]+$/ - Само букви (без цифри и символи).
  • /\S+@\S+\.\S+/ - Много проста проверка за имейл (нещо @ нещо . нещо).
const phoneRegex = /^\d{10}$/; // Търси точно 10 цифри
const phoneNumber = "0888123456";

if (phoneRegex.test(phoneNumber)) {
    console.log("Валиден телефон!");
} else {
    console.log("Невалиден формат.");
}

Интерактивен пример (Комбиниран)

Тук комбинираме всичко: HTML атрибути, Статична проверка при бутона и Динамична проверка за паролата.

Името трябва да е поне 3 символа.

Моля, въведете валиден имейл.

Паролите не съвпадат или са празни!

• Поне 6 символа

• Поне една цифра

Успех!

Данните са валидни и готови за изпращане.

Време за практика!

Кликнете върху всяка задача, за да видите условието.

1. Задължително поле (HTML)
Създайте проста форма с едно поле за име и бутон. Добавете атрибута required в HTML-а и пробвайте да я изпратите празна.
2. Проверка на дължина (Статична)
Създайте поле за "Коментар". При submit, проверете с JS дали дължината е над 10 символа. Ако не е, използвайте alert() и event.preventDefault().
3. Валидация в реално време (Динамична)
Изберете едно поле (напр. "Потребител"). Добавете слушател за input събитието. Ако потребителят въведе "admin", оцветете рамката в червено (защото е заето име), иначе - в зелено.
4. Телефонен номер (Regex)
Направете поле за телефон. При изпращане проверете дали съдържа само цифри (/^\d+$/.test(value)). Ако има букви, покажете грешка.
5. Сравняване на пароли
Класическа задача: Две полета за парола. При промяна на второто поле (събитие input), проверявайте дали съвпада с първото. Ако не - покажете червен текст "Не съвпадат".
6. Чекбокс "Съгласен съм"
Добавете чекбокс. При опит за изпращане, проверете свойството .checked. Ако е false, спрете формата и оцветете текста на чекбокса в червено.
7. Бутон "Show Password"
Добавете бутонче до полето за парола. При клик, сменете атрибута type на полето от "password" на "text" и обратно.
8. Деактивиране на бутона
Направете бутона за изпращане disabled по подразбиране. Активирайте го с JS само когато всички полета са попълнени валидно.
